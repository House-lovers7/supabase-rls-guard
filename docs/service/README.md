# Supabase Migration SQL・RLSスポット監査

Supabase プロジェクトのマイグレーション SQL（最大50ファイル）を対象に、OSS ツール
[supabase-rls-guard](../../README.md) の静的スキャンと人手レビューを組み合わせて、
Row Level Security の設定ミスを洗い出すスポット監査サービスです。

## 提供内容

1. supabase-rls-guard 最新版による静的スキャン（RLS001–RLS018）
2. 検出結果の人手判定（真陽性 / 偽陽性 / 意図的公開の切り分けと優先度付け）
3. 監査報告書の納品（[報告書テンプレート](./templates/audit-report.md) 準拠）
4. 修正後の再確認 1回（初回納品から30日以内。報告書記載の指摘への対応確認）

## 価格

- 10,000円/時間（税別）
- 作業時間は最大5時間まで。上限金額は 50,000円（税別）
- 想定作業時間を超えそうな場合は、超過前に必ず合意を取り直します

## 範囲外（含まれないもの）

- 修正実装（報告書の推奨対応を実装する作業は範囲外です。必要な場合は別途相談）
- ライブデータベースに対する監査（本サービスは静的解析のみ）
- RLS 以外を含む包括的なセキュリティ診断・ペネトレーションテスト
- 継続的な監視・定期監査（スポット1回 + 再確認1回のみ）

## 前提と透明性

- 本サービスは静的解析と人手レビューによる「既知の、静的に検出可能なミス」の
  洗い出しです。報告書は対象システムに問題が存在しないことを示すものでは
  ありません。
- 2026年7月時点で有償監査実績は0件です。初期のご依頼はその前提をご理解の上で
  お申し込みください。粒度と形式は
  [合成サンプル報告書](./samples/synthetic-audit-report.md) で事前に確認できます。
- お預かりした SQL・関連資料の取り扱い（保持・削除期限）は
  [運用ランブック](./operator-runbook.md) に従います。顧客資料を公開リポジトリに
  入れない境界は
  [ADR-0001](../03_adrs/ADR-0001-rls-spot-audit-service-boundary.md)
  で固定しています。

## 申し込みの流れ

1. **15分の適合確認**（無償）: 対象が本サービスに適合するか確認します
   （マイグレーション SQL が最大50ファイル以内か、静的監査で目的を満たせるか）。
2. 適合する場合、[発注・受け入れテンプレート](./templates/order-and-intake.md)
   に沿って対象と条件を確定します。
3. 受領 → スキャン → 人手判定 → 報告書納品 →（任意）再確認。
