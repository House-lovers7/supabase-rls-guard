# ADR-0001: RLSスポット監査サービスと OSS リポジトリの境界

- 状態: 採用
- 日付: 2026-07-19

## 文脈

supabase-rls-guard は公開 OSS リポジトリである。その上に有償の
RLSスポット監査サービス（docs/service/）を載せるが、顧客から受領する
マイグレーション SQL・zip・非公開報告書は秘密情報であり、公開リポジトリに
混入してはならない。

## 決定

- 顧客資料（受領 SQL、zip、非公開報告書、作業ファイル）はすべて
  **リポジトリ外**で管理する。作業の都合でリポジトリ直下に置く場合も
  Git 管理外の `.audit-work/` に限定し、`.gitignore`（`.audit-work/` /
  `audit-work/` / `*.zip` / `*-private-report.md`）で強制する。
- リポジトリに含めるのは、オファー（docs/service/README.md）、テンプレート、
  **合成サンプル**（examples 由来）のみ。実案件の成果物・顧客名は含めない。
- 保持・削除の期限（再確認 30日 / 完全削除 37日）は
  docs/service/operator-runbook.md に固定する。
- サービス文書の整合性（オファー・価格・上限・免責・誇大表現の禁止）は
  `tests/service-docs.test.ts` の契約テストで固定し、CI で検証する。

## 結果

- 公開リポジトリから顧客情報が漏れる経路を構造的に断てる。
- 文書更新時の価格・範囲の不整合や誇大表現の混入を CI が検出する。
- ツール本体（静的解析 CLI）とサービス（人手判定・納品運用）の責任境界が
  文書とテストで明示される。
